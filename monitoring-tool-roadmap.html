Self-Hosted Kubernetes Monitoring Tool

A Go + React observability platform that uses Prometheus as the metrics source of truth, adds dashboards, auth, live updates, and alerting, then deploys into the AlmaLinux K3s cluster.

Architecture Flow

K3s

Cluster Workloads

Nodes, pods, kubelet, cAdvisor, node exporter, and kube-state-metrics expose metrics.

->
Prometheus

Metrics Store

Scrapes and stores time-series data. Remains the only raw metrics database in v1.

->
Go

Backend API

Auth, PromQL proxy, dashboard APIs, alert APIs, WebSocket live updates, and health checks.

->
Postgres

App Database

Stores users, dashboards, panels, alert rules, notification channels, and alert history.

->
React

Frontend

Operator UI for dashboards, charts, PromQL editing, alerts, and cluster overview.

Key rule: the backend stores product data, not raw time-series metrics.

  • Templates
